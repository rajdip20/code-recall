Exercise: Working With Structured Data

  1. Download and open the "Working With Structured Data - Starter.xlsx" file (it's at the bottom of this page, like usual).

Exercise 1

  1. On the "Exercise 1" worksheet, use Excel's Text to Columns feature to split the data in column A into distinct columns. HINT: you'll need to use a different delimiter than a comma!

Exercise 2

  1. Download and open the "External Data Files.zip" file at the bottom of this page.

  2. Within that zip file, open the "External Data 1.csv" file.

  3. Save the file AS "External Data.csv" to some location on your computer (the specific folder doesn't matter, just remember it for the next step!).

  4. Using Excel's menu for working with external data, import the contents of the "External Data.csv" file into the "Working With Structured Data - Starter.xlsx" file.

  5. Name the resulting Excel Table "home_sale_data".

Exercise 3

  1. Open again the "External Data Files.zip" file and go to the External Data 2.csv" file within

  2. Save the file over the .csv file you saved in Exercise 2 - in other words, the name of the saved file should be "External Data.csv".

  3. Take note of how many rows of data are currently in the "home_sale_data" table.

  4. Refresh the data source for the the "home_sale_data" table. Does the number of rows in the table change?

Exercise 4

  1. In the "home_sale_data" table, insert a blank column to the right of the "SalePrice" column. Give the column the name "AgentCommission".

  2. Populate this new column with a formula that returns 5% of the number in the "SalePrice" column (i.e., multiplies it by 0.05). Make sure your formula is "table style", referencing table columns instead of cell references.

Exercise 5

  1. Navigate to the "Regions" worksheet; convert the range of values in columns A and B to an Excel table. Name the table "regions".

  2. Back in the "home_sale_data" table, insert a blank column to the right of the "City" column. Give the column the name "Region".

  3. Keying off the values in the "City" column, use a VLOOKUP function to pull the region values from your newly-created lookup table into the blank "Region" column.

  4. Scan the values in the "Region" column for errors - you will probably notice some for records where the city is "Lexington".

  5. Go back to the "Regions" worksheet, and add the following values at the bottom of the table:

  6. "Lexington", in the "City" column

  7. "Central", in the "Region" column

  8. Reassess the "Region" column in the "home_sale_data" table - are the errors still there?

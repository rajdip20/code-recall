Exercise: VBA Macros

Part 1

Even if you skipped the "Excel 101" section of the course, take a moment to review the exercises at the end of that section. We're going to be automating most of these steps with a VBA macro!

After acquainting yourself with those steps, download and open the "Macros - Starter.xlsx" file that's at the bottom of this page.

Part 2

Within the "Macros - Starter.xlsx" file, record a macro (named "ProcessData") to capture the following actions:

  1. Delete the "ProductID" column in the data.

  2. Unmerge the two cells composing the "CustomerName" column header, and then add "Customer First Name" and "Customer Last Name" headers for the two columns.

  3. Cut the "Customer First Name" and "Customer Last Name" columns, and insert them to the left of the "ReviewStars" column.

  4. Edit the remaining column headers to have spaces between the words in the names (i.e, "ProductName" becomes "Product Name").

  5. Insert a new column, to the left of the other columns, and give it a column header of "Review Number".

  6. Using Excel's autofill feature, populate this column with a series of numbers from one to the number of reviews in the dataset.

  7. Re-format the "Purchase Date" and "Review Date" columns to the MM/DD/YYYY date format.

  8. Re-format the "Review Stars" column to display zero digits after the decimal place.

  9. Using Excel's find and replace feature, replace all instances of the word "ukelele" on the worksheet with "ukulele".

  10. Change the font of all the cells in the range to "Segoe UI", size 10.

  11. Give the column headers a dark background fill color from one of Excel's built-in "theme" colors - your choice!

  12. Change the font color of the column headers to white.

  13. Bold the column headers.

  14. Auto-fit the widths of the columns to adjust to the width of the data they contain.

  15. Freeze the top row of the data (the column headers) so that it remains visible no matter how far you scroll down through the data.

  16. Finally, stop recording the macro.

Part 3

Edit the macro code as follows:

  1. Fix any missteps you made while recording (if any), that Excel captured in the VBA code.

  2. Remove any extraneous lines of VBA code the were generated by actions you don't need to capture.

  3. Have the macro name the "Review Number" column "Review ID" instead.

  4. Add a line of code at the beginning of the macro that suppresses alerts/warnings/dialog boxes for the duration of the macro.

  5. Add a line of code at the end of the macro that causes a message box to appear after the macro completes, which says: "Congratulations, your macro ran successfully!".

  6. Save the file as "Customer Reviews Template.xlsm" (note the macro-enabled file format). This is now your template file.

Part 4

  1. Delete all the columns containing data on the worksheet.

  2. Unfreeze panes in the worksheet.

  3. Add a button to the worksheet, and assign it to your "ProcessData" macro.

  4. Give the button a caption of "Execute Process".

  5. Make sure the button is positioned far enough to the right to allow room for us to copy and paste in new data.

  6. Save the template file (don't want to lose your changes!).

Part 5

Time to put your template file to the test!

  1. Re-open the "Macros - Starter.xlsx" file in the resources section below.

  2. Copy and paste the data from the "Macros - Starter.xlsx" file into cell A1 of our template file.

  3. Click your "Execute Process" button, and hope for the best!

  4. If anything goes awry, edit the macro code and see if you can diagnose the issue.
